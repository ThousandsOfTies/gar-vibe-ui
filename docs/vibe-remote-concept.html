1. ビジョン

解きたい課題

AIエージェントは作業の途中で、人間の判断を必要とする。PCの前へ戻って状態を確認するだけの時間は、思ったより流れを止める。

本質的に必要なのは、状態を知ること、必要なら承認・却下すること、短い追加指示を返すこと。

体験の理想形

ソファや別作業中でも、VS Codeビューや手元の小さなデバイスで、エージェントが今どこにいるかを確認する。

画面には「作業中か」「待っているか」「どのエージェントが報告しているか」が短く出る。物理ボタンは次フェーズの入力導線になる。

UXは維持。 変わったのは実装方式です。VS Codeの他プラグインを外側から押すのではなく、対応エージェントがMCP/ブリッジへ状態を自己申告し、デバイスはその状態を購読します。

① Agent Cooperation

Codexや各社エージェントが、作業中・待機中・完了・失敗をMCPツールで明示的に報告する。

② Status Hub

Vibe Remoteが状態と小型デバイス向けUIを保持し、VS Code内ビューや物理デバイスへ配信する。

③ Physical UX

M5StickC Plus2では状態表示、device UI表示、A/B/Pボタンからの選択返却まで実機確認済み。

3. 技術方針の変更

項目旧方式新方式
状態取得VS Code拡張が外側から推定MCP対応エージェントが自己申告
承認待ち他プラグインのUIやコマンドに依存エージェントが承認要求をブリッジへ出す
OK/NGVS Codeコマンド中継で押すdevice UIのactionとして表示し、物理ボタン入力をuiActionとして返す
対応範囲VS Code内部実装に左右されるMCP/ブリッジ対応したエージェントで安定
やらないこと： VS CodeのDOMや他拡張の非公開状態を無理に操作しない。これはプラグインごとに壊れやすく、汎用リモコンとして成立しにくい。

4. MCPステータスブリッジ

現行実装では、エージェントが次のMCPツールで状態を書き込めます。

Tool用途
vibe_remote_set_statusrunning, waiting, done, failed, idle を明示的に設定
vibe_remote_heartbeat作業継続中のハートビートを更新
vibe_remote_request_decision人間の判断待ちと選択肢を表示し、structuredContent.ui_id を返す
vibe_remote_show_ui小型デバイスへタイトル、メッセージ、最大3件の項目とアクションを表示し、structuredContent.ui_id を返す
vibe_remote_get_actionデバイスから返るボタンアクションを取得。既定では最大60秒待機
vibe_remote_clear_ui表示中のdevice UIをクリア
vibe_remote_clear_status状態をidleへ戻す

6. デバイス連携

PC側設定

vibeRemote.bindAddress = 0.0.0.0
vibeRemote.discoveryEnabled = true
Vibe Remote: サーバを再起動
Vibe Remote: 接続トークンを表示

M5Stack側

cd vibe-remote/m5stickc-client
pio run
pio run -t upload
pio device monitor

Wi-Fi SSID、パスワード、接続トークンは .env.local または platformio.ini の build_flags で指定する。

現在地： M5StickC Plus2でmDNS探索、WebSocket接続、device UI表示、A/B/PボタンによるuiAction返却まで実機確認済み。UIが無い時は状態表示と簡易status送信にフォールバックする。

7. デバイス表示

LCD表示例

┌──────────────────┐
│ WAITING           │
│ Decision          │

表示ルール

Status cardstate と title を表示。タイトルは実機上では9文字まで。
Messagefields が無ければ最大3行で折り返し。fields がある場合は1行に省略。
Fields最大3行。message がある場合は残り2行、無い場合は3行。labelは7文字、valueは18文字まで。
FooterA / B / P のアクションを常時表示。

ボタン対応

Front shortA
Side shortB
Power shortP
Front holdA-hold
Side holdB-hold
Power holdP-hold

状態色

青 — done

黄 — waiting / decision needed

緑 — running / working

赤 — failed / disconnected
